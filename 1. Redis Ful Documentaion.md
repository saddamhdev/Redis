# **1. What is Redis?**

Redis is an **in-memory NoSQL data store**, meaning:

* All active data stays in **RAM (very fast)**
* Redis can also store data on **disk** using persistence
* Redis is ideal for caching, login systems, rate limiting, pub/sub, leaderboards, etc.

---

# **2. In-Memory vs Disk Persistence**

### **RAM (In-Memory):**

* Super fast
* Data disappears if server restarts
* Used for login, sessions, caching, real-time operations

### **Disk Persistence (Permanent Storage):**

Redis has two persistence types:

| Persistence                   | What It Does                   |
| ----------------------------- | ------------------------------ |
| **RDB** (Redis Database File) | Saves snapshot every X seconds |
| **AOF** (Append Only File)    | Logs every write operation     |

â¡ With RDB/AOF **no data loss happens** even after server restart.

---

# **3. Redis Persistence â€” How It Works**

### **Live Operation**

* Redis reads/writes data from **memory**
* Very fast (microseconds)

### **Server Restart**

* Redis loads data **from RDB/AOF file into memory**

### **Key Point**

âœ” Redis NEVER reads from disk during normal operations
âœ” Disk is used only to **restore data on restart**
